NSURLSession部分方法

一. NSURLSession

-(NSURLSessionDataTask )dataTaskWithRequest:(NSURLRequest )request completionHandler:(void (^)(NSData data, NSURLResponse response, NSError *error))completionHandler`

  • - dataTaskWithRequest:completionHandler:
    • 第一个参数:请求对象
    • 第二个参数:completionHandler 当请求结束(成功|失败)的时候调用
    • data:响应体
    • response:响应头信息
    • error:错误信息
    • 注意:block在子线程中执行

-(NSURLSessionDataTask )dataTaskWithURL:(NSURL )url completionHandler:(void (^)(NSData data, NSURLResponse response, NSError *error))completionHandler

  • - dataTaskWithURL:completionHandler:
    • 说明:该方法内部会自动将url包装成一个请求对象|GET请求

-(NSURLSessionDataTask )dataTaskWithRequest:(NSURLRequest )request completionHandler:(void (^)(NSData data, NSURLResponse response, NSError *error))completionHandler

  • - dataTaskWithRequest:completionHandler:
    • 创建检索基于指定的URL请求对象上的URL的内容的任务，并调用完成时的处理程序。

-(void)resume

  • 启动任务,执行任务

-(void)cancle

  • 取消任务

-(void)suspend

  • 暂停任务

Creating a Session 创建一个会话

+(NSURLSession )sessionWithConfiguration:(NSURLSessionConfiguration )configuration delegate:(id)delegate delegateQueue:(NSOperationQueue *)queue

  • + sessionWithConfiguration:delegate:delegateQueue:
    • 创建指定的会话配置，委托和操作队列的 会话。

二. NSURLSessionTask

  • NSURLSessionTask
    • NSURLSessionDataTask
      • NSURLSessionUploadTask
    • NSURLSessionDownloadTask

Adding Download Tasks to a Session 在会话中添加一个下载任务

-(NSURLSessionDownloadTask )downloadTaskWithURL:(NSURL )url completionHandler:(void (^)(NSURL location, NSURLResponse response, NSError *error))completionHandler

  • 创建一个下载任务检索指定URL的内容，结果保存到一个文件中，并调用完成后的处理程序。
    • 接受到数据之后内部会直接写入到沙盒里面(子线程)

-(NSURLSessionDownloadTask )downloadTaskWithResumeData:(NSData )resumeData

  • 创建一个下载任务，恢复先前取消或失败的下载。

Canceling a Download 取消下载

-(void)cancelByProducingResumeData:(void (^)(NSData *resumeData))completionHandler

  • 取消下载并调用恢复数据以备后用的一个回调。

三. Delegate Methods

  1. NSURLSessionDelegate
  2. NSURLSessionTaskDelegate
  3. NSURLSessionDataDelegate
  4. NSURLSessionDownloadDelegate

3. NSURLSessionDataDelegate

-(void)URLSession:(NSURLSession )session dataTask:(NSURLSessionDataTask )dataTask didReceiveResponse:(NSURLResponse *)response completionHandler:(void (^)(NSURLSessionResponseDisposition disposition))completionHandler

  • 当接受到服务器响应的时候调用

-(void)URLSession:(NSURLSession )session dataTask:(NSURLSessionDataTask )dataTask didReceiveData:(NSData *)data

  • 接受到服务器返回数据的时候调用

-(void)URLSession:(NSURLSession )session task:(NSURLSessionTask )task didCompleteWithError:(NSError *)error

  • 告诉代理 任务完成数据传输。这个方法当请求完成的时候调用

4. NSURLSessionDownloadDelegate

-(void)URLSession:(NSURLSession )session downloadTask:(NSURLSessionDownloadTask )downloadTask didWriteData:(int64_t)bytesWritten totalBytesWritten:(int64_t)totalBytesWritten totalBytesExpectedToWrite:(int64_t)totalBytesExpectedToWrite

  • 写数据(接收数据之后调用)
    • bytesWritten:本次写数据的大小
    • totalBytesWritten:已经写入数据的大小
    • totalBytesExpectedToWrite:要下载文件的总大小

-(void)URLSession:(NSURLSession )session downloadTask:(NSURLSessionDownloadTask )downloadTask didResumeAtOffset:(int64_t)fileOffset expectedTotalBytes:(int64_t)expectedTotalBytes

  • 当恢复下载的时候调用

-(void)URLSession:(NSURLSession )session downloadTask:(NSURLSessionDownloadTask )downloadTask didFinishDownloadingToURL:(NSURL *)location

  • 当下载完成的时候调用
    • 内部已经实现了边下载边写入数据到沙盒的操作
    • downloadTask内部时可以拿到响应头信息
      • 例如:
      • downloadTask.response.suggestedFilename: 响应头的建议名称

-(void)URLSession:(NSURLSession )session task:(NSURLSessionTask )task didCompleteWithError:(NSError *)error

  • 整个请求结束的时候调用
